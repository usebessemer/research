# v1.6 — Scope (retroactive)

**Status:** Backfilled 2026-05-23 after v1.6 shipped. Lightweight — v1.6 was hygiene-focused; this file exists primarily to establish the format for v1.7+.

## Research question

Can routine repo-hygiene work be reliably executed by a Sonnet-class agent given small, well-defined issue scopes and green-test exit criteria?

## Why this matters now

Establishes baseline operating pattern: a higher-context agent scopes the issue, a Sonnet-class agent executes, a human approves the merge. v1.6 is a low-stakes proving ground for that loop before v1.7's bigger scope.
